Строка в go имеет следующую структуру:

type _string struct {
    elements *byte // указатель на массив байт 
    len      int   // длина 
}

В кишках строки находится массив из байт
Строка всегда имеет длину, но у нее нет емкости, т.к. строки иммутабельны 
Когда используем конкатенацию, мы получаем совершенно новую строку - байтовый массив копируется в другое место с учетом новых изменений 

Символы английского алфавита кодируются одним байтом, а вот к примеру на иероглифы может уходить 2 или 3 байта 
Поэтому длина строки не всегда равна количеству символов
